# 反愿景构建法

## R — 原文 (Reading)

> 现在,我们要把这些转化成一种我称之为"反愿景"的东西,也就是,对你绝对不想过的那种人生有一个清晰而残酷的认知。这样,你就能利用这种负能量,把自己的努力指向积极方向,从内在动机出发行动。
>
> — 作者, 第6节

---

## I — 方法论骨架 (Interpretation)

**反愿景**是对"绝对不想过的生活"的清晰认知。人对"不想要什么"的判断往往比"想要什么"更清晰。

为什么用负能量:
- 正向目标常常模糊、不够动力
- 描述糟糕的未来更容易、更具体
- "害怕失去"比"想要得到"更有驱动力

如何构建反愿景:
1. 描述5年后一个普通的星期二——最糟糕版本
2. 描述10年后——错过什么机会、谁对你失望
3. 临终时——"我选择了最安全的版本,从未打破任何模式"的代价
4. 写一句话反愿景压缩版,读到时应有感触

## A2 — 触发场景 (Future Trigger) ★

### 用户会在什么情境下需要这个 skill?

1. 用户说"我不知道自己想要什么"
2. 用户对正向目标没动力
3. 用户说"定目标但总是完不成"
4. 用户说"没有想要的东西"
5. 用户对"愿景"这类词无感

### 语言信号 (用户的话里出现这些就应激活)

- "我不知道想要什么"
- "没有动力"
- "不知道活着的意义"
- "迷茫"
- "什么都不想要"

### 与相邻 skill 的区分

- 与 `愿景` 的区别: 反愿景是"不想要",愿景是"想要"
- 与 `身份层级` 的区别: 反愿景是动力工具,身份层级是改变层面

## E — 可执行步骤 (Execution)

当 skill 被激活后, agent 应按以下步骤执行:

1. **确认用户状态**
   - 问:"你现在是不知道想要什么,还是知道但不行动?"
   - 完成标准: 用户能回答

2. **引导描述5年后最糟糕的星期二**
   - 问:"想象5年后一个普通的星期二,是你最不想过的版本。告诉我:"
     - 你在哪里醒来?
     - 身体感觉如何?
     - 早上9点到下午6点你在做什么?
     - 晚上10点你的感受?
   - 完成标准: 用户能具体描述

3. **引导描述10年后**
   - 问:"10年后,你错过了什么?谁对你失望了?"
   - 完成标准: 用户能列出具体条目

4. **引导临终视角**
   - 问:"想象你的人生走到尽头。你选择了最安全的版本,从未打破任何模式。那代价是什么?"
   - 完成标准: 用户能说出代价

5. **写反愿景压缩版**
   - 问:"用一句话概括你绝不允许自己的人生变成什么样"
   - 完成标准: 用户写出一句话,读到时有感触

6. **转化到正向**
   - 问:"既然这样,你想要什么?"
   - 完成标准: 用户开始描述想要��

## B — 边界 (Boundary) ★

### 不要在以下情况使用此 skill

- 用户已有清晰目标,只是在问方法
- 用户处于抑郁状态(需要专业帮助)
- 用户拒绝思考负面

### 作者在书中警告的失败模式

- **把目标当终点**: "等我减完肥,我就可以好好享受生活了"——负能量没有转化

### 作者的盲点 / 时代局限

- 高估了负能量的持续性
- 需要配合正向愿景才能持续

### 容易混淆的邻近方法论

- 恐惧驱动: 只是害怕,而反愿景是用害怕来明确方向
- 负面思维: 只看到负面,而反愿景是转化负能量
