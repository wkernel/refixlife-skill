# Refixlife 智能技能路由器

## R — 原文 (Reading)

> 我们不是要教用户成为各种方法的专家，而是要帮他们快速找到解决当前问题的最合适工具。
> 
> — 作者, 综合应用章节

## I — 方法论骨架 (Interpretation)

Refixlife 是一个技能路由系统，其核心思想是：
1. 用户通常能描述痛苦点但不知道该用什么方法解决
2. 不同的问题状态对应不同的解决方法论
3. 通过关键语言信号快速匹配最合适的技能
4. 一次性提供完整的处理方案而不仅是单一工具

触发规则：
- 用户说不知道想要什么 → 触发 anti-vision (反愿景构建法)
- 用户说努力但没效果 → 触发 identity-level + cybernetics (身份层级 + 控制论)
- 用户说想彻底改变 → 触发 one-day-reset (一天重置法)
- 用户说坚持太苦 → 触发 gamification (游戏化)
- 用户说迷茫我是谁 → 触发 mind-stage (心智阶段)

## A1 — 书中的应用 (Past Application)

### 案例: 一站式问题解决流程

- **问题**: 用户描述混乱的困扰状态
- **方法论的使用**: 通过问题分类匹配对应技能组合
- **结论**: 提供精准的处理方案
- **结果**: 用户快速获得针对性指导

## A2 — 触发场景 (Future Trigger) ★

### 用户会在什么情境下需要这个 skill?

1. 用户说"我不知道想要什么"或类似表达
2. 用户说"我努力了但没有效果"
3. 用户说"我想彻底改变自己"
4. 用户说"坚持下去太痛苦了"
5. 用户说"我迷茫，我不知道自己是谁"

### 语言信号 (用户的话里出现这些就应激活)

**反愿景触发信号:**
- "我不知道想要什么"
- "没有目标"
- "不知道要追求什么"
- "对正向提升没动力"

**身份层级+控制论触发信号:**
- "我努力了但没效果"
- "不管怎么做都没用"
- "付出和回报不成正比"
- "感觉自己在原地踏步"

**一天重置触发信号:**
- "我想彻底改变"
- "从今天开始重新做人"
- "想要全新的开始"
- "厌倦了现在的生活"

**游戏化触发信号:**
- "坚持太苦了"

### 与相邻 skill 的区分

- 与单一 skill 的区别: 这是一个分发器，而非具体方法
- 与 一般咨询 的区别: 提供具体可执行的技能而不仅是建议

## E — 可执行步骤 (Execution)

当 skill 被激活后, agent 应按以下步骤执行:

1. **问题倾听与定位**
   - 仔细聆听用户描述的困扰
   - 识别关键情绪词和问题核心
   - 完成标准: 能概括出用户的主要困扰点

2. **信号匹配与技能选择**
   - 根据触发规则匹配对应技能
   - 多重匹配时提供技能组合建议
   - 完成标准: 明确确定要触发的 skill 组合

3. **技能激活与执行引导**
   - 按顺序激活匹配到的 skill
   - 为每个 skill 提供执行入口和使用指引
   - 完成标准: 用户能开始使用对应技能进行自我反思

4. **后续跟踪建议**
   - 建议用户在使用技能后的反思时间
   - 提供效果评估的简单方法
   - 完成标准: 用户知道如何判断技能是否有效

5. **组合使用指导**
   - 解释不同 skill 如何协同工作
   - 建议使用顺序和时间间隔
   - 完成标准: 用户了解如何组合使用多个技能

## B — 边界 (Boundary) ★

### 不要在以下情况使用此 skill

- 用户已经明确要求使用某个具体 skill
- 用户处于危机状态需要专业心理干预
- 用户只是想了解技能理论而不想实践
- 用户对所描述的问题状态没有真实体验

### 作者在书中警告的失败模式

- **过度依赖路由**: 把路由当作最终解决方案而不执行具体技能
- **错误匹配**: 因为表面相似而匹配错误的技能
- **忽视组合效应**: 只使用单一技能而忽略可能的协同效果

### 作者的盲点 / 时代局限

- 低估了用户自我诊断的难度
- 高估了用户在情绪困扰时的理性判断能力
- 需要结合情感陪伴才能更有效

### 容易混淆的邻近方法论

- 问题清单: 只是列问题而不提供解决路径
- 方法大全: 提供所有方法而不做筛选
- 咨询式建议: 给建议而不提供具体可执行的框架
