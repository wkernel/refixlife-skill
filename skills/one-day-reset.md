# 一天重塑流程

## R — 原文 (Reading)

> 这个流程可以在一天内能完成的结构:早上,你会做一场"心理挖掘",把你那些隐藏很深的动机翻出来。白天,你会用一些"中断提示"逼自己跳出自动驾驶模式,重新思考你的生活。晚上,你会把所有洞见整合成一个清晰方向。
>
> — 作者, 第6节

---

## I — 方法论骨架 (Interpretation)

这是一套每年用一次的"人生重置"流程,需要一整天(约8小时)。核心是通过深度自我质疑,把潜意识中的动机翻到明面上,打破无意识自动驾驶模式。

**早上(15-30分钟)心理挖掘**:
- 回答关于"你不爽但忍着的事"、"反复抱怨但不改的事"的问题
- 从行为推断真实的潜意识欲望
- 写出你无法向尊敬的人坦白的真相

**白天(随时)中中断提示**:
- 在手机设置随机时间的中断提醒
- 问题如:"我现在做的这件事,是在回避什么?""我是在走向我讨厌的生活,还是我想要的生活?"

**晚上(30-60分钟)综合洞见**:
- 写出真正的敌人(内在模式/信念,不是外部环境)
- 写出反愿景压缩版和最小可行愿景
- 制定一年/一月/每日视角的目标

## A1 — 书中的应用 (Past Application)

### 案例 1: 作者的临界点突破

- **问题**: 作者人生中最好的阶段,都发生在被"自己毫无进展"逼到彻底受不了之后
- **方法论的使用**: 他是被动达到临界点,但这个流程让你主动创造临界点
- **结论**: 需要积累足够的"不协调"才能触发改变
- **结果**: 这个流程帮助用户先进入不协调临界点

## E — 可执行步骤 (Execution)

当 skill 被激活后, agent 应按以下步骤执行:

1. **确认用户有足够时间**
   - 问:"你有一整天(8小时)的时间吗?"
   - 判停条件: 若没有,建议分拆成多个半天

2. **引导早上心理挖掘**
   - 提供问题清单,让用户写下来:
     - 有什么事情你一直不爽但忍着?
     - 过去一年你反复抱怨但不改的事是什么?
     - 如果有人只看你的行为,会认为你想要什么?
   - 完成标准: 用户能写出真实答案(不是表面答案)

3. **引导反愿景构建**
   - 问:"5年后你最糟糕的生活是什么样的?10年后呢?"
   - 完成标准: 用户能具体描述

4. **引导���小可行愿景**
   - 问:"如果你要打个响指三年后过上想要的生活,会是什么样子?"
   - 完成标准: 用户能描述出"最想要的生活"

5. **设置白天中断提示**
   - 建议在手机设置6个随机时间提醒
   - 提供6个问题让用户随时思考

6. **引导晚上综合**
   - 问:"真正的敌人是谁?"(不是外部,是内在模式)
   - 问:"用一句话概括你最不想成为的人"
   - 完成标准: 用户写出一句话

7. **目标制定**
   - 一年视角: 什么必须发生才能打破旧模式?
   - 一月视角: 让年度目标可能发生,需要什么结果?
   - 每日视角: 这周2-3个时间块行动是什么?
   - 完成标准: 用户写出三层目标

## B — 边界 (Boundary) ★

### 不要在以下情况使用此 skill

- 用户没有整块时间(需要8小时)
- 用户只是想做简单反思,不需要深度
- 用户已经有清晰方向,只是在问执行方法

### 作者在书中警告的失败模式

- **只有被逼到绝境才改变**: "我人生中最好的阶段,往往都发生在我被自己毫无进展逼到彻底受不了之后"
- **只做表面文章不断重复旧模式**: "坦白说,如果你不打破那些让你停在原地的无意识模式,改变是不可能发生的"

### 作者的盲点 / 时代局限

- 高估了一天流程的效用——真正的改变需要持续行动
- 假设用户有足够时间和意愿

### 容易混淆的邻近方法论

- 普通日记: 只记录,不挖掘深层动机
- 年度计划: 关注要做的事,不关注为什么做
