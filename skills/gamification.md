# 人生游戏化框架

## R — 原文 (Reading)

> 你的愿景就是你获胜的方式。你的反愿景代表你可能失去什么。你的1年目标就是你的主线任务。你的1月项目就像打最终大怪,通过它你才能升级、拿战利品。你的每日行动就是日常任务。每天的流程会不断为你解锁新机会。你的约束就是游戏规则。这些约束反而会激发你的创造力。
>
> — 作者, 第7节

---

## I — 方法论骨架 (Interpretation)

把人生要素转化为游戏元素:**

- **愿景** = 获胜方式(最终胜利)
- **反愿景** = 失败代价(失败惩罚)
- **1年目标** = 主线任务(游戏的核心目标)
- **1月项目** = 升级打怪(通过后才能升级)
- **每日行动** = 日常任务(每天做的,积累经验)
- **约束** = 游戏规则(限制反而激发创意)

为什么有效:
- 游戏让人痴迷,因为有清晰目标、即时反馈、成长感
- 把"苦差事"变成"有趣的事"
- 约束反而激发创造力

**六要素结构**:
1. 反愿景
2. 愿景
3. 一年目标
4. 一月项目
5. 每日杠杆
6. 底线约束

## A1 — 书中的应用 (Past Application)

### 案例: 游戏的心流体验

- **问题**: 改变是苦差事,难以坚持
- **方法论的使用**: 把人生当游戏,用游戏机制驱动
- **结论**: 游戏让人沉浸,因为有痴迷、心流、清晰
- **结果**: "我们就可以让生活变得更享受、更专注和更成功"

## A2 — 触发场景 (Future Trigger) ★

### 用户会在什么情境下需要这个 skill?

1. 用户说"改变太难了"
2. 用户说"坚持不下来"
3. 用户说"过程太无聊"
4. 用户说"没有动力"

### 语言信号 (用户的话里出现这些就应激活)

- "太难了"
- "坚持不下来"
- "无聊"
- "没动力"

### 与相邻 skill 的区分

## E — 可执行步骤 (Execution)

当 skill 被激活后, agent 应按以下步骤执行:

1. **确认用户状态**
   - 问:"你现在是对改变 没兴趣,还是觉得太难?"
   - 完成标准: 用户能回答

2. **介绍游戏化概念**
   - 解释每个要素如何对应游戏元素
   - 完成标准: 用户理解框架

3. **帮助用户建立六要素**
   - 反愿景: "你最无法忍受的生活是什么?"
   - 愿景: "你想要的生活是什么?"
   - 一年目标: "一年后你想达成什么?"
   - 一月项目: "这个月你要突破什么?"
   - 每日杠杆: "每天哪2-3件事最重要?"
   - 底线约束: "什么是你绝对不愿牺牲的?"
   - 完成标准: 用户能写出六要素

4. **强化游戏感**
   - 告诉用户: "你是玩家,人生是游戏"
   - 问:"如果你在玩游戏,你会放弃吗?"
   - 完成标准: 用户调整心态

5. **建立注意力保护**
   - 告诉用户: "这六要素会形成��心力场,保护你的注意力"
   - 完成标准: 用户理解保护机制

## B — 边界 (Boundary) ★

### 不要在以下情况使用此 skill

- 用户对"游戏化"概念反感
- 用户已有强内在动机
- 用户处于危机状态

### 作者在书中警告的失败模式

- **把目标当终点**: "不要担心所谓的终点,因为正如你将会发现的,它根本不存在"

### 作者的盲点 / 时代局限

- 游戏化可能过于简化复杂的人生问题
- 不是所有人都喜欢游戏

### 容易混淆的邻近方法论

- 普通目标管理: 只关注目标,不关注"好玩"
- 奖励系统: 外在激励,而游戏化是内在转化
