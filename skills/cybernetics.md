# 控制论反馈系统

## R — 原文 (Reading)

> 控制论说明了智能系统的特性：拥有一个目标。朝目标行动。感知自己当前的位置。将其与目标进行比较。再根据反馈继续行动。你可以根据一个系统在反复试错中迭代和坚持的能力,来判断它的智力。
>
> — 作者, 第5节

---

## I — 方法论骨架 (Interpretation)

**控制论**来自希腊语"操舵",是"获得你想要之物的艺术"。

智能系统的反馈循环:
1. **目标**: 明确你要什么
2. **行动**: 朝目标行动
3. **感知**: 知道自己当前在哪
4. **比较**: 对比目标和现状
5. **迭代**: 根据反馈调整,继续行动

**高智力** = 能从这个循环中学习并坚持
**低智力** = 卡在问题上,遇到阻碍就放弃

关键洞察:
- 错误不是失败,而是反馈
- 放弃是因为把错误当作终点而不是信息
- 时间尺度足够大,任何问题都能解决

## A1 — 书中的应用 (Past Application)

### 案例: 作家建立读者群

- **问题**: 一个作家无法建立读者群,因为缺乏尝试新方法、实验以及找出适合自己的流程的能力
- **方法论的使用**: 把失败当作信息而非终点,继续迭代
- **结论**: "认为自己无法创造出有效流程这种想法本身就是可被证伪的"
- **结果**: 持续迭代直到找到有效流程

## A2 — 触发场景 (Future Trigger) ★

### 用户会在什么情境下需要这个 skill?

1. 用户说"为什么我总是做不好"
2. 用户说"我尝试了但没用"
3. 用户在经历失败,问"要不要放弃"
4. 用户说"我不知道自己做得好不好"
5. 用户在制定学习/成长计划

### 语言信号 (用户的话里出现这些就应激活)

- "为什么做不好"
- "试过没用"
- "要不要放弃"
- "不知道进步了没有"
- "怎么判断对不对"

### 与相邻 skill 的区分

- 与 `试错` 的区别: 控制论强调系统化的反馈,非盲目试错

## E — 可执行步骤 (Execution)

当 skill 被激活后, agent 应按以下步骤执行:

1. **确认用户是否有明确目标**
   - 问:"你想达成什么?"
   - 判停: 若无目标,先帮助明确目标

2. **检查用户处在这个循环的哪一步**
   - 问:"你现在是卡在行动上,还是不知道自己在哪?"
   - 完成标准: 用户能回答

3. **帮助建立反馈机制**
   - 问:"你怎么知道自己做得怎么样?"
   - 提供: 量化指标、同伴反馈、结果检验等
   - 完成标准: 用户有一个可感知的方式

4. **重新定义"错误"**
   - 告诉用户: 错误是信息,不是失败
   - 问:"这次失败告诉了你什么?"
   - 完成标准: 用户能从错误中提取信息

5. **建立迭代思维**
   - 问:"下次你会怎么做?"
   - 完成标准: 用户能说出调整方案

6. **提醒时间尺度**
   - 告诉用户: "只要时间尺度足够大,任何问题都能被解决"
   - 完成标准: 用户理解需要坚持

## B — 边界 (Boundary) ★

### 不要在以下情况使用此 skill

- 用户没有目标,只是在抱怨
- 用户处于危机状态(需要情感支持)
- 用户只是在问信息,不涉及行动

### 作者在书中警告的失败模式

- **低智力:无法从错误中学习**: "低智力的人会卡在问题上,而不是去解决它们。他们遇到阻碍就放弃"
- **以为无法创造出有效流程**: "认为自己无法创造出有效流程这种想法本身就是可被证伪的"

### 作者的盲点 / 时代局限

- 假设用户有足够的耐心和时间
- 未考虑资源限制

### 容易混淆的邻近方法论

- 意志力方法: 以为靠坚持就能成功
- 线性规划: 关注计划,不关注反馈
